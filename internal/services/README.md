# Services Layer

## 역할
도메인 객체와 포트를 조합해 유즈케이스를 오케스트레이션한다. "사용자의 한 가지 행동이 일으키는 비즈니스 흐름"의 실행 단위를 담당한다.

## 책임
- 유즈케이스 흐름 제어 (입력 검증, 도메인 호출, 포트 호출 순서 조립)
- 트랜잭션·일관성 경계 관리
- 유즈케이스 입출력 DTO 정의

## 의존 방향
`domain`에만 의존한다. 트랜스포트(HTTP/WS)와 인프라 구현은 알지 못한다. `interfaces`에서 호출되며, 인프라 구현은 포트를 통해 주입받는다.

## 용어 메모 (FE 시점)
- **유즈케이스(Use Case)** — 사용자의 한 가지 행동이 일으키는 비즈니스 흐름 한 단위. FE의 redux thunk나 react-query mutation 하나에 대응.
- **포트(Port)** — 이 레이어가 외부 자원(저장소, 외부 시스템 등)을 사용할 때 호출하는 인터페이스. 구현은 인프라 레이어가 제공.
- **DTO** — 레이어 경계를 넘기는 데이터 전용 객체. FE의 API request/response 타입과 같은 역할.
- **트랜잭션 경계** — 여러 동작이 "전부 성공 또는 전부 실패"해야 하는 묶음의 범위.
- **트랜스포트(Transport)** — 외부와의 통신 채널(HTTP, WebSocket 등). FE에서 `fetch`/WebSocket을 다루는 부분.
