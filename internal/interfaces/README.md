# Interfaces Layer

## 역할
외부 프로토콜(HTTP, WebSocket 등)의 요청을 유즈케이스 호출로 변환하는 어댑터. 외부 세계와 애플리케이션 사이의 경계 역할을 한다.

## 책임
- 요청 파싱과 입력 검증, 외부 표현 → 내부 DTO 매핑
- 유즈케이스 호출과 결과의 직렬화
- 프로토콜 단위 에러 변환 (예: HTTP 상태 코드, WebSocket 클로즈 코드)

## 의존 방향
`services`와 `domain`에 의존한다. 인프라 구현의 세부에는 의존하지 않으며, 인프라 자원이 필요할 때도 서비스/포트를 통해서만 접근한다.

## 용어 메모 (FE 시점)
- **어댑터(Adapter)** — 외부의 프로토콜·형식을 내부 호출 형태로 변환해 주는 구현. FE에서 fetch 응답을 앱이 쓰는 타입으로 매핑하는 부분과 같은 역할.
- **유즈케이스(Use Case)** — 사용자의 한 가지 행동이 일으키는 비즈니스 흐름 단위. 이 레이어는 외부 요청을 받아 해당 유즈케이스를 실행한다.
- **DTO** — 레이어 경계를 넘기는 데이터 전용 객체. FE의 API request/response 타입과 같은 역할.
- **트랜스포트(Transport)** — HTTP, WebSocket처럼 외부와의 통신 채널. 이 레이어가 다루는 영역.
