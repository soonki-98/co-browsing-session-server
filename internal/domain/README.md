# Domain Layer

## 역할
비즈니스 규칙과 외부 레이어가 구현할 포트(인터페이스)를 정의한다. 시스템의 가장 안쪽 레이어이며, 어떤 기술 선택과도 무관한 추상으로 유지된다.

## 책임
- 엔티티 / 값 객체로 도메인 개념과 불변식 모델링
- 도메인 규칙과 행위 표현
- Repository를 비롯한 포트 인터페이스 정의

## 의존 방향
어떤 레이어에도 의존하지 않는다. 다른 모든 레이어가 이 레이어에 의존한다. 외부 라이브러리·프레임워크에 대한 의존도 두지 않는다.

## 용어 메모 (FE 시점)
- **엔티티 / 값 객체** — 도메인 모델 객체. FE의 store에 들어가는 model 타입과 비슷하지만, 식별자와 행위(메서드)를 함께 가진다. 값 객체는 식별자 없이 값 자체로 비교되는 불변 객체.
- **포트(Port)** — "이런 기능이 필요해"를 선언하는 인터페이스. 구체 구현은 다른 레이어가 제공한다. FE의 `interface ApiClient` 같은 추상 계약.
- **Repository** — 저장소(DB/캐시/메모리) 접근을 추상화한 포트. FE의 `userApi.getById()` 같은 API 클라이언트와 같은 위치이지만, 백엔드에서는 저장소 추상화에 해당.
