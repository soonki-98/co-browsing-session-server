# App (Composition Root)

## 역할
컴포지션 루트. 모든 레이어의 구체 구현을 포트 인터페이스에 바인딩해 의존성을 조립하고 애플리케이션을 부팅한다.

## 책임
- 구체 타입 선택과 의존성 주입 (어떤 Repository 구현, 어떤 핸들러를 쓸지)
- 설정 로딩과 각 컴포넌트로의 주입
- 애플리케이션 부팅과 라이프사이클(시작·종료) 와이어링

## 의존 방향
모든 레이어에 의존한다. 단, 비즈니스 로직은 두지 않는다 — 오직 "무엇을 어떻게 연결할지"만 다룬다.

## 용어 메모 (FE 시점)
- **컴포지션 루트(Composition Root)** — 앱 시작 지점에서 모든 의존성을 조립해 트리에 주입하는 곳. React의 `App.tsx`에서 Provider/Store를 합성해 루트에 꽂는 것과 같은 역할.
- **의존성 주입(DI)** — 의존 객체를 직접 만들지 않고 밖에서 받아 쓰는 것. FE의 `Context.Provider`로 hook이 의존성을 받는 패턴과 유사.
