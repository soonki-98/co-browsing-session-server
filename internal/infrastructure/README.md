# Infrastructure Layer

## 역할
`domain`이 정의한 포트의 기술적 구현을 제공한다. 저장소·외부 시스템 같은 기술 디테일을 이 레이어 안에 격리한다.

## 책임
- Repository 등 포트의 구체 구현 (메모리/외부 스토어 등)
- 외부 시스템(타 서비스, 메시지 브로커 등)과의 통신 어댑터
- 직렬화, 락, 커넥션 관리 같은 기술 디테일 처리

## 의존 방향
`domain`에만 의존한다. `services`/`interfaces`로부터 직접 import 되지 않으며, `app`이 포트 인터페이스에 이 레이어의 구현을 바인딩한다.

## 용어 메모 (FE 시점)
- **포트(Port) / 어댑터(Adapter)** — 포트는 "이런 기능이 필요해"라는 인터페이스, 어댑터는 그 구체 구현. 이 레이어가 작성하는 코드는 거의 모두 어댑터다. FE로 치면 `interface ApiClient`(포트) ↔ `class AxiosApiClient`(어댑터)의 어댑터 쪽.
- **Repository** — 저장소 접근을 추상화한 포트. 이 레이어에서는 그 포트의 실제 구현(메모리, DB 등)을 제공한다.
